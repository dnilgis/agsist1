Accessibility Statement

Last updated: April 2026

Commitment

AGSIST is built and maintained by one person with the goal of being usable by everyone — including people with disabilities. Accessibility isn't an afterthought here; it's part of the build standard on every page.

Standards

AGSIST targets WCAG 2.1 Level AA compliance. Every page in the site is built with:

  • A skip-to-main-content link at the top of every page
  • Sufficient color contrast in dark mode (the primary interface)
  • Full keyboard navigation support
  • Descriptive alt text on all images
  • ARIA labels on interactive elements
  • Proper landmark regions (main, nav, footer)
  • Responsive layout that works at all zoom levels
  • touch-action: manipulation on all interactive elements to prevent iOS double-tap delay
  • Minimum font size of .62rem enforced across all components

Known Limitations

Some real-time data tables and embedded charts (radar, drought monitor) have limited screen reader support. These are identified as areas actively being improved. No content on this site is exclusively communicated through color alone.

Feedback

If you run into an accessibility barrier on AGSIST, please contact me directly. I'm a solo developer and I take these reports seriously — it's the fastest way to get something fixed.
